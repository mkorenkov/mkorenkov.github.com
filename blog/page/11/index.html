Documents automation solution

Apr 21st, 2011

Пришло время опять делать разные договоры и я всячески оттягивают этот момент. Вхлам надоело делать это “руками”, а поскольку никто не предложил решения для Documents automation idea, придется думать самому.

Прежде всего, шаблоны MS Office и Open Office меня не устраивают - я не хочу быть зависимым от платформы и меня не устраивает решение, в котором в результате работы я получу груду файликов. Если мастер-копия документов будет доступна только на 1 десктопе, это затруднит передачу процесса другому исполнителю.

В общем, руководствуясь невнятными требованиями, я стал рассматривать остальные инструменты под рукой. В Google Docs я не нашел ничего связанного с шаблонами (может, искал плохо). И обратил взор на Atlassian Confluence, в котором оказалась нужная функциональность.

Раскурив шаблоны, вооружившись знанием и получим правильные привилегии - Space Admin, задача сдвинулась с мертвой точки. К концу первого дня у меня были все необходимые шаблоны основных документов. В принципе, если бы я отнесся к задаче с большим уровнем осознанности, я бы усмотрел очевидный риск, - печать документов. Мало сгенерить на основании шаблона wiki страничку, которая выглядит правильно, необходимо ее получить на печать в аутентичном виде.

Доступный инструмент Confluence экпорта в PDF повел себя совсем неадекватно: не печатал кириллицу. Экспорт в word давал на выходе документ со странным форматированием и без приатаченых картинок - как-то их забывал.

Текущий план: использовать export в HTML и допиливать специальным bash или python скриптом. Спасибо всем, кто дочитал до конца. Продолжение следует.

Ubuntu 10.10 and ZTE AC 8710

Apr 20th, 2011

Переодически я настраиваю PeopleNet для Ubuntu. Для версий, которые младше 10.10, я пользовался старым постом onorua. Сегодня, после привычной modprobe (ниже) Network manager подпрыгнул и сказал, что готов настроить для меня Mobile broadband связь.

modprobe usbserial vendor=0x19d2 product=0xfffe

В списке уже был PeopleNet, что давало определенную надежду. Однако из коробки он не завелся. В процессе я опробовал CrossPlatformUI-V1.0.27-BSNL-i386-ubuntu и несколько менее экзотических способов.

В финале, ZTE 8710 и Ubuntu удалось скрестить путем установки нового Network Manager из ppa (https://launchpad.net/~network-manager/+archive/ppa).

Я по-старинке прописал в /etc/apt/sources.list:

 

deb http://ppa.launchpad.net/network-manager/ppa/ubuntu maverick main 

deb-src http://ppa.launchpad.net/network-manager/ppa/ubuntu maverick main

Выполнил:

 

sudo apt-key adv –keyserver keyserver.ubuntu.com –recv-keys BC8EBFE8

sudo aptitude update

sudo aptitude upgrade

И вуаля, Network Manager в Ubuntu 10.10 теперь может использовать мой 3g-модем.

Осталось прописать modprobe usbserial vendor=0x19d2 product=0xfffe в /etc/rc.local, чтобы после перезагрузки все продолжало работать.

 

 

 

 

Irresponsible deb packages removal

Apr 20th, 2011

В процессе инсталляции CDMA-модема, я установил deb-пакет CrossPlatformUI-V1.0.27-BSNL-i386-ubuntu. К сожалению, этот пакет не решил мою проблему, при установке выдал невнятную ошибку и не стал работать. Удалить его тоже оказалось проблемой: ни визуальные средства, ни apt-get с dpkg не смогли помочь. В большинстве случаев я получал ошибку:

I wasn’t able to locate file for the crossplatformui package. This might mean you need to manually fix this package. 

Для решения проблемы пришлось поправить пост-ремув скрипт и первой инструкцией впаять exit через

cd /var/lib/dpkg/info/

sudo gedit crossplatformui.postrm
